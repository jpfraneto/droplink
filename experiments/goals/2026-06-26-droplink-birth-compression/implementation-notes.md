# Implementation Notes

## Near-Term Code Changes

1. Normalize SKU into the fulfillment spec.

Current type:

```ts
export type RelicFulfillmentSpec = {
  catalogProductId: number;
  catalogVariantId: number;
  // no first-class SKU
};
```

Suggested addition:

```ts
sku?: string;
```

Parse it from Printful variant raw data using common keys:

```ts
function variantSku(raw: Record<string, unknown>) {
  const value = raw.sku ?? raw.catalog_variant_sku ?? raw.variant_sku;
  return typeof value === "string" && value.trim() ? value.trim() : undefined;
}
```

2. Add variant-level catalog candidates for planning.

Current `printfulCatalogOptionsForPlanning` emits product-level options:

```ts
{ key, name, type, placements }
```

Experiment version should emit:

```ts
{
  candidateId,
  catalogProductId,
  catalogVariantId,
  sku,
  productName,
  variantName,
  productType,
  placement,
  technique,
  roleAffinity
}
```

3. Split final prompt construction into a small compiler.

Move product prompt assembly out of `generateDropFromUrl` into testable functions:

```ts
buildProductPrintPrompt({ brand, study, plan, relic, selection, page })
buildGroupProductPrompt({ brand, study, plan, relics, selectedProducts })
```

4. Persist prompt metadata.

For each prompt, persist:

```text
promptVersion
mechanism
selectedProduct
requiredMentions
negativeConstraints
sourceEvidenceHash
```

5. Add a validator before image generation.

Block image generation until:

```text
3 selected products valid
3 print prompts valid
group prompt valid
prompt safety valid
```
